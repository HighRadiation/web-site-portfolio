# Yazılan Prompt
Vercel / Next.js derleme (build) sırasında alınan TypeScript tip hatası.

# Yapılan Plan
1. **Derleme Hatasının Analizi**: Next.js üretilen üretim (production) derlemesinde alınan hatanın sebebi:
   - Supabase Postgrest sorgularının döndürdüğü `PostgrestFilterBuilder` tipinin standard `Promise<T>` tipini tam olarak genişletmemesi, sadece `PromiseLike<T>` (thenable) olması.
   - `safeQuery` tanımında `promise` argümanının tipi `Promise<T>` olarak tanımlandığından, Postgrest sorgusu geçildiğinde tip uyuşmazlığı yaşanıyordu.
   - Ek olarak, `fallback` argümanının tipi `T` (yani Postgrest'in ham dönüş tipi) olarak beklendiği halde bizim `{ count: 0 }` gibi daha sade fallback objeleri geçmemiz derleme hatasına yol açıyordu.
2. **Hata Düzeltme**:
   - `safeQuery` fonksiyon imzasındaki `promise` tipi standard `PromiseLike<T>` yapıldı.
   - `fallback` için ayrı bir jenerik tip parametresi `F` tanımlanarak (`safeQuery = async <T, F>(promise: PromiseLike<T>, fallback: F): Promise<T | F>`), veri yapılarının birbirine katı biçimde bağlanması ve derleyicinin hata vermesi engellendi.
3. **Doğrulama ve Derleme**:
   - Değişiklikler sonrasında lokalde `npm run lint` ve `npm run build` komutları çalıştırıldı. Derleme %100 sıfır hata ile başarıyla tamamlandı.
4. **Belgeleme**: Yapılan işlemler bu dokümantasyon dosyasına kaydedildi.
