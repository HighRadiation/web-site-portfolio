# Single-Page Portfolio UI Oluşturulması

## Yazılan Prompt
Ek'teki UI tasarımına göre tek sayfalık bir portfolio sitesi oluştur.
Navbar linkleri ayrı sayfalara değil, aynı sayfadaki section'lara scroll etsin.
Mevcut CSS temasını koru.

## Yapılan Plan
1. globals.css → tamamen yeniden yazıldı (tema korunarak genişletildi)
2. Navbar → anchor link'li (#home, #about, vb.) tek sayfa navigasyonu
3. page.tsx → section component'lerinden oluşan tek sayfa
4. 5 yeni section component oluşturuldu
5. Footer sadeleştirildi
6. Layout sadeleştirildi

## Yapılan İşlem

### Yeni Dosyalar
- `src/components/sections/Hero.tsx` — Hero section (badge, başlık, CTA butonları)
- `src/components/sections/About.tsx` — Hakkımda (metin + code block)
- `src/components/sections/Timeline.tsx` — Experience & Education (timeline)
- `src/components/sections/Projects.tsx` — Proje kartları (3 sütun grid)
- `src/components/sections/Contact.tsx` — İletişim (centered CTA)

### Güncellenen Dosyalar
- `src/app/globals.css` — Tamamen yeniden yazıldı (purple accent, mono font, animasyonlar)
- `src/app/page.tsx` — Section component'leri import ediyor
- `src/app/layout.tsx` — Sadeleştirildi (Source_Serif_4 kaldırıldı)
- `src/components/layout/Navbar.tsx` — Anchor link navigasyonu
- `src/components/layout/Footer.tsx` — Sadeleştirildi

### Tema Özellikleri
- Dark background (#1e1e1e)
- Purple accent (#7c3aed)
- Mono font (JetBrains Mono) butonlarda ve badge'lerde
- Smooth scroll (CSS scroll-behavior)
- Sticky navbar (backdrop-blur)
- Timeline ile dot animation
- Project card hover efektleri

### Sonuç
- `npm run lint` → 0 hata
- `npx tsc --noEmit` → 0 hata
- Dev server çalışıyor, tüm section'lar görünüyor
