## Yazılan Prompt

> Tutarlılık
> 4. Login server action'ına Zod ekle

## Yapılan Plan

Projelerdeki diğer admin formlarının (projeler, mesajlar, yetenekler vs.) server action'ları `useActionState` ile entegre bir biçimde Zod doğrulaması (`ActionState` yapısı üzerinden) gerçekleştiriyor. Login sayfası ise sadece URL'deki `?error=` search params üzerinden çalışıyordu ve Next.js'in standart server function'ıydı. Bu tutarsızlığı gidermek için login server action'ının Zod ile doğrulanması hedeflendi.

**Strateji:**
1. `src/lib/validations/auth.ts` dosyası oluşturulup email ve password için `loginSchema` Zod şeması tanımlanacak.
2. `src/app/[locale]/login/actions.ts` içindeki `login` metodu `ActionState` dönecek şekilde güncellenip Zod `safeParse` eklenecek.
3. `src/app/[locale]/login/page.tsx` sayfası `use client` ile Client Component'e çevrilerek `useActionState` eklenecek. Aynı zamanda URL error (mevcut uyumluluk için) veya Zod'dan dönen `fieldErrors` mesajları UI üzerinde gösterilecek.

## Yapılan İşlem

### Eklenen Dosya
**`src/lib/validations/auth.ts`**
- `loginSchema` eklendi (Email geçerliliği ve minimum 6 karakter şifre kısıtı ile).

### Değiştirilen Dosyalar

**`src/app/[locale]/login/actions.ts`**
- `login` metodu `(prev: ActionState, formData: FormData) => Promise<ActionState>` imzasıyla güncellendi.
- Doğrulama hatası varsa URL redirect yerine `fieldErrors` ile `ActionState` nesnesi dönmesi sağlandı.

**`src/app/[locale]/login/page.tsx`**
- Bileşen Client Component (`'use client'`) haline getirildi.
- Next 15 `useActionState` hook'u sisteme entegre edildi ve hata dönüldüğünde hatalı input'ların altına Zod'dan dönen kırmızı hata mesajları eklendi.
- Input kenarlıklarının (border) hata durumunda kırmızı olması (`borderColor: '#ef4444'`) sağlandı.
- Bileşeni sarmalaması için `next/navigation`'dan `useSearchParams` eklendi ve `<Suspense>` ile dışa aktarıldı.
