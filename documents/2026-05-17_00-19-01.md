## Yapılan Plan

Scalar API dokümantasyon sayfası (`/docs`) ve arka planda veri sunduğu OpenAPI json dosyası şeması rotası (`/api/docs`) arasındaki koruma tutarsızlığını gidermek için kullanıcıdan Seçenek A (Tam koruma) onayı alındı.

**Sorun:**
- `/docs` yolu middleware tarafından korunarak yetkisiz erişimlerde `/login` sayfasına yönlendiriliyordu.
- Ancak, middleware yapısındaki `/api` bypass kuralından dolayı `/api/docs` rotası tamamen korumasızdı. Yetkisiz kişiler doğrudan bu JSON şemasına erişip projedeki tüm API uçlarını görebiliyordu.

**Strateji:**
- `/api/docs` rotasının (`src/app/api/docs/route.ts`) içerisine Supabase sunucu istemcisi entegre edilecek.
- İstek atan kullanıcının oturum bilgisi `supabase.auth.getUser()` ile kontrol edilecek.
- Oturum açmamış istekler doğrudan `401 Unauthorized` durum kodu ile reddedilerek iki rota arasındaki güvenlik açığı kapatılacak.

## Yapılan İşlem

### Değiştirilen Dosyalar

**`src/app/api/docs/route.ts`**
- Supabase sunucu istemcisini oluşturmak için `createClient` import edildi.
- `GET` fonksiyonunda `supabase.auth.getUser()` ile kullanıcı kontrol edildi.
- Eğer kullanıcı nesnesi bulunmuyorsa, `NextResponse.json({ error: 'Unauthorized' }, { status: 401 })` dönecek şekilde güvenlik doğrulama adımı eklendi.
- Yetkili kullanıcı durumunda ise OpenAPI JSON içeriği başarıyla döndürülmeye devam ediyor.
